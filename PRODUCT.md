# Product

## Register

product

## Users

Residents submit city repair requests and track progress. Administrators review, assign, and monitor work orders. Maintenance workers accept and complete assigned orders. Course reviewers inspect whether the system is runnable, explainable, and supported by collaboration evidence.

## Product Purpose

The product demonstrates a complete vertical information-system workflow for a course assignment: database design, REST APIs, frontend-backend interaction, testing, GitHub collaboration, and AI-assisted development logs. Success means the team can run the system, complete the repair-order loop in 5 minutes, and trace each feature to code, data, tests, and evidence.

## Brand Personality

Clear, restrained, trustworthy. The interface should feel like a practical public-service management tool rather than a decorative marketing site.

## Anti-references

Do not make a tech-showcase big screen, neon dashboard, glassmorphism UI, landing page, or unrelated smart-city platform. Avoid purple gradients, excessive motion, decorative cards, and invented workflows that do not support the course scope.

## Design Principles

1. Keep the business loop visible and easy to demonstrate.
2. Prefer stable, familiar product UI patterns over novelty.
3. Make status, ownership, and next actions obvious for each role.
4. Preserve evidence: every feature should connect to an API, database record, test, screenshot, or AI log.
5. Optimize for team consistency so separate modules feel like one system.

## Accessibility & Inclusion

Use readable contrast, keyboard-focusable controls, clear validation messages, and reduced-motion-friendly transitions. The prototype should remain usable on common laptop screens.

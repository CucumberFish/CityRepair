# 发给组员的统一通知

大家先不要直接写代码。按下面步骤完成开发准备，完成后在群里回复自己的分支名、Skill 读取结果、数据库/API 是否理解。

## 第一步：拉代码和开分支

```bash
git clone <GitHub仓库地址>
cd <仓库目录>
git checkout develop
git pull origin develop
git checkout -b feat/<自己的分支名>
git push -u origin feat/<自己的分支名>
```

- 苟佳豪：`feat/resident-repair`
- 闫龙飞：`feat/admin-dispatch`
- 袁翊博：`feat/worker-processing`
- 孔祥洋：`feat/dashboard-category`
- 张天童：`feat/evaluation-quality`

## 第三步：发给自己的 AI

把下面提示词发给你正在使用的 AI：

```text
你现在参与《城市报修与工单处理系统》开发。

请先读取并遵守：
1. README.md
2. docs/00-开发启动包索引.md
3. docs/数据库与接口定稿.md
4. docs/GitHub协作与提交规范.md
5. 城市报修与工单处理系统-团队分工与Agent-Skills开发规范.md
6. .agents/skills/city-repair-project-rules/SKILL.md

如果涉及前端页面，还必须读取：
.agents/skills/city-repair-ui-style/SKILL.md

在编码前先回复：
- 我负责的模块边界；
- 我的分支名；
- 我允许修改的文件范围；
- 我会使用的数据库表；
- 我会实现或调用的接口；
- 我不能做的事情；
- 我需要真实执行的测试。

在我确认前，不要修改代码。
```

## 第四步：开发要求

- 只能改自己模块；
- 不新增工单状态；
- 不改数据库表名和核心字段；
- 不新建第二套登录/权限系统；
- 不写死用户 ID；
- 不只靠前端隐藏按钮做权限；
- 每次提交只解决一个明确问题；
- 至少保留 3 次有效 commit。

## 第五步：提交前必须交

使用模板：

- `templates/AI协作日志模板.md`
- `templates/测试记录模板.md`
- `templates/PR自检模板.md`

没有真实命令、截图或接口测试证据的，不能合并。
